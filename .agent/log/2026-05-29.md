# 2026-05-29

## Cosserat-solver extraction audit (no code changes)

Scoped how to lift YarnBall's Cosserat rod solver out for use in NVIDIA
Isaac Lab (massively-parallel RL, ~1000 rod-per-arm envs, max throughput).
Pure analysis — **no source or build changes this session.**

Audited the sim internals; the durable, repo-relevant findings went to
`NOTES.md ## Architecture quirks`:

- collision is one global-LBVH path — self-collision == inter-rod
  collision, no self-only toggle (`detectCollisions` + `buildCollisionList`);
- contact response is fused inside `cosseratItr` (runs ×numItr), detection
  is separate (×1) — solver dominates;
- `bvh.query()` forces a CPU↔GPU sync per detection step (`d_flags[0]`);
- N independent rods already coexist in one flat array delimited by
  `flags=0` (createFromCurves) → batchable in one kernel sweep when
  collision is off;
- the physics TUs use no GL symbols (refined the existing
  "no headless library variant" note — the split is more contained than
  it looked).
